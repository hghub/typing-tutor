# Marriage Decision System Brief

Working name:
- `Marriage Readiness & Match Guide`

Alternative public names:
- `Nikah Decision Guide`
- `Spouse Compatibility Guide Pakistan`
- `Marriage Clarity System`

Status:
- `Do not build as a normal calculator`
- `Do not publish without expert review`

## 1. Purpose

This system should help a Pakistani Muslim user think more clearly about:
- readiness for marriage
- suitability of a specific match
- questions that still need answers
- red flags they may be minimizing
- whether to proceed, slow down, pause, or walk away

It must not act like:
- a fatwa engine
- a “marry this person” machine
- a simplistic compatibility score

## 2. Guiding Ethos

The system should be shaped by:
- Quranic emphasis on taqwa, truthfulness, chastity, justice, mercy, consultation, and responsibility
- Shia scholarly emphasis on religion, akhlaq, compatibility, reason, honesty, and family environment
- natural human realities: attraction, temperament, conflict style, boundaries, money, family systems, and safety

Important product stance:
- the tool should be `Quran-inspired in values`
- but it should `not claim divine certainty`
- it should be `firm against red flags`
- and `humble about what software cannot know`

## 3. High-Risk Nature

This is a sensitive domain. Harm can happen if the system:
- pushes false certainty
- shames users
- rewards performative religiosity over character
- ignores abuse/control patterns
- becomes a vehicle for class/tribe prejudice
- lets users weaponize “Islamic scoring” against others

So the system must prefer:
- clarification over verdict
- evidence over fantasy
- safety over romantic momentum
- responsibility over pressure

## 4. Decision Model

The system should use 4 output pillars:

1. `Foundation`
- deen seriousness
- honesty
- character
- responsibility
- modesty/chastity

2. `Compatibility`
- communication style
- emotional maturity
- conflict approach
- family expectations
- gender-role expectations
- education/work expectations
- children expectations
- sect/practice alignment

3. `Risk`
- anger
- secrecy
- controlling behavior
- disrespect
- financial irresponsibility
- manipulation
- family toxicity
- addiction
- unresolved past issues

4. `Clarity`
- how much is known directly
- what is assumption vs evidence
- what must still be verified

The system should never reduce all of this to a single number.

## 5. Primary Modes

### A. Marriage Readiness
Purpose:
- tell the user whether they themselves are ready for marriage

Focus:
- emotional steadiness
- financial realism
- ability to take responsibility
- conflict maturity
- family-boundary readiness
- intention and seriousness

### B. Match Evaluation
Purpose:
- help evaluate one specific potential match

Focus:
- character and deen
- compatibility
- known risks
- missing information

### C. Comparison Mode
Purpose:
- compare two possible directions carefully

Constraint:
- do not rank people theatrically
- compare fit/risk patterns, not “who is better”

## 6. Question Architecture

The flow should be multi-step.

### Step 1: Intention and Context
- arranged / semi-arranged / self-found
- user gender
- age bracket
- urgency level
- family pressure level
- city / migration context
- sect/practice context

### Step 2: My Own Readiness
- why do I want marriage now?
- what do I expect from marriage?
- can I handle disagreement respectfully?
- what kind of life am I actually prepared to build?
- am I stable enough to carry responsibilities?

### Step 3: Core Foundations of the Match
- religious seriousness
- truthfulness
- modesty/chastity boundaries
- dependability
- respectfulness
- sense of duty

### Step 4: Compatibility
- communication
- emotional style
- family involvement
- work/education expectations
- money expectations
- lifestyle rhythm
- relocation and mobility
- children expectations
- social comfort and class expectations

### Step 5: Red Flags
- temper / volatility
- mockery / contempt
- secretive behavior
- financial chaos
- controlling language
- pressure to rush
- dishonesty
- unresolved addiction
- history of boundary violation
- use of religion to silence questions

### Step 6: Clarity Gaps
- what have you seen directly?
- what comes from family reputation only?
- what has not yet been discussed openly?
- what needs neutral verification?

## 7. Output Logic

Preferred result language:
- `Strong foundation, but more clarity needed`
- `Good surface fit, but risk is too high`
- `Religious alignment is present, but emotional maturity looks weak`
- `Proceed slowly and ask these questions before commitment`
- `Do not rush this match`

Never say:
- `Marry this person`
- `This person is ideal`
- `Islam says yes`

## 8. Red-Flag Engine

If certain answers cluster together, the system should increase caution sharply.

Examples:
- high pressure + low transparency + anger
- strong family dominance + weak boundary respect
- religious language + dishonest actions
- attraction high but clarity low
- urgency high but readiness weak

When severe patterns appear, output should shift from “optimize” to “protect”.

## 9. Sensitive Pakistan-Specific Realities

This system must understand:
- biradari pressure
- parental control
- migration expectations
- working-woman expectations
- provider anxiety
- in-law boundary stress
- social image pressure
- “good family” claims without real behavior evidence

The tool must be anti-naive, but not anti-family.

## 10. Faith-Informed Principles to Encode

Without presenting itself as a fatwa engine, the system should quietly weight:
- taqwa over glamour
- akhlaq over status
- trustworthiness over performance
- compatibility over fantasy
- knowledge over assumption
- consultation over impulsiveness

Primary source themes to keep in design review:
- piety / moral seriousness
- purity and chastity
- truthfulness
- justice and mercy
- consultation and knowledge
- not following what one does not know

## 11. Unsafe Outputs to Prevent

The tool should explicitly avoid:
- validating stalking or secret investigation
- turning ethnicity/tribe/class into hard filters
- rewarding wealth over character
- encouraging marriage despite clear control/abuse signs
- presenting “religious appearance” as enough
- claiming certainty where information is incomplete

## 12. Expert Review Requirements

Do not publish until reviewed by:
- `1 Shia scholar or qualified teacher`
- `1 Muslim marriage counselor / psychologist`
- `2-3 Pakistani users in realistic marriage-search contexts`

Review questions:
- is this faithful in values?
- is this fair to both men and women?
- does it minimize harm?
- does it overclaim?
- can an emotionally vulnerable user misuse it?

## 13. Premium Boundary Later

Possible premium features:
- saved match profiles
- compare two match profiles privately
- export a discussion checklist
- deeper family-boundary module
- pre-engagement conversation planner

## 14. Build Order

1. Write final question bank
2. Write scoring + caution rules
3. Write safe-output rules
4. Build readiness mode first
5. Build match-evaluation mode second
6. Run expert review before launch

## 15. Non-Negotiable Product Rule

This tool must help users become:
- more truthful
- more careful
- more self-aware
- less impulsive

If it only makes them more confident without making them wiser, it has failed.
