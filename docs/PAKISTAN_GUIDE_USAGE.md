## Pakistan-Friendly Guide Usage

Use the in-page Pakistan-friendly guide only when it improves clarity for real users. Do not add it mechanically to every tool.

### Full Guide

Use the full guide when the tool is:
- high-stakes
- jargon-heavy
- not obvious from the title alone
- useful but under-discovered
- strongly Pakistan-first in language or workflow

Full guide includes:
- `Simple explanation`
- `Common need`
- Roman Urdu search phrases

Current full-guide candidates:
- `investment-allocation-planner`
- `rent-vs-buy-pakistan`
- `salary-offer-evaluator`
- `freelance-tax-planner`
- `loan-emi`
- `data-leak-detector`
- `text-extractor`
- `doc-redaction`
- `voice-invoice`

### Light Guide

Use the light guide when the tool is already mostly understandable, but a short plain-language bridge still helps.

Light guide includes:
- `Simple explanation` only

Current light-guide candidates:
- `solar-planner`
- `tax-calculator`
- `salary-slip`
- `gold-price`
- `budget-splitter`
- `urdu-keyboard`
- `whatsapp-tools`

### No Guide

Do not add the guide when the tool is already obvious and the extra block would feel repetitive or forced.

Typical no-guide tools:
- `word-counter`
- `age-calculator`
- `unit-converter`
- `qr-generator`
- other simple utility tools with self-explanatory names

### Decision Rule

Ask:
1. Will a first-time Pakistani user understand the tool from the title alone?
2. Is the user likely to search in Roman Urdu or think in Urdu first?
3. Is the tool something many people would benefit from but may not know exists?
4. Is the decision sensitive enough that a short plain-language bridge reduces misuse?

If the answer is strongly yes for 3 or 4, use `full`.
If the answer is yes for 1 or 2 only, use `light`.
If the tool is already obvious, use `none`.

### Writing Rules

- Keep it short and useful.
- Do not stuff keywords.
- Use Roman Urdu only where it sounds natural.
- Avoid making the block feel like hidden SEO.
- Prefer everyday phrasing over formal translation.

### Review Cadence

Review these guide blocks:
- when Search Console shows new Roman Urdu query patterns
- when a tool gets more mainstream and no longer needs the full block
- when a tool changes purpose enough that the simple explanation becomes stale
